# RAGAS 自动化评测报告：V1.0 vs V2.0

## 一、RAGAS 评测简介

RAGAS（Retrieval Augmented Generation Assessment）是专门用于评估 RAG 系统的自动化框架，提供了比传统指标更专业的评测维度。

### 核心指标说明

1. **Faithfulness (忠实度)**
   - 含义：答案是否基于检索到的文档，而不是模型自己编造
   - 反幻觉指标：分数越高，幻觉越少
   - 范围：0-1，1 表示完全忠实于文档

2. **Answer Relevancy (答案相关性)**
   - 含义：答案是否切题，是否回答了用户的问题
   - 范围：0-1，1 表示完全相关

3. **Context Precision (上下文精度)**
   - 含义：检索到的文档中，有多少是真正相关的
   - 衡量检索质量：分数越高，检索越精准
   - 范围：0-1，1 表示所有检索文档都相关

4. **Context Recall (上下文召回)**
   - 含义：所有相关文档中，检索到了多少
   - 衡量检索覆盖率：分数越高，遗漏越少
   - 范围：0-1，1 表示检索到了所有相关文档

## 三、评测结果

| Context Precision (上下文精度) | 0.217 | ❌ 较差 |
| Context Recall (上下文召回) | 0.400 | ⚠️ 中等 |

| Answer Relevancy (答案相关性) | N/A | ❌ 评测失败 |
| Context Precision (上下文精度) | 0.200 | ❌ 较差 |

| **Context Recall** | 0.400 | 0.400 | 0.000 (0%) | ➡️ 保持不变 |

## 四、关键发现

### ✅ 改进点

**1. 忠实度提升 13%**
- V2.0 的混合检索让答案更加基于文档内容
- 减少了模型自己编造信息的情况
- 这是最重要的改进，直接提升了答案可信度

### ⚠️ 需要注意的问题

**1. 上下文精度较低（0.2 左右）**
- 检索到的文档中，只有 20% 真正相关
- 说明检索到了很多无关文档
- 可能原因：
  - 知识库文档质量不高
  - 检索参数（k=4 或 k=6）设置过大
  - 文档切分不合理

**2. 上下文召回中等（0.4）**
- 只检索到了 40% 的相关文档
- 说明还有 60% 的相关信息被遗漏
- 可能原因：
  - 知识库不完整
  - 检索参数 k 设置过小
  - Embedding 模型效果不够好

**3. Answer Relevancy 评测失败**
- 技术原因：`OpenAIEmbeddings` 接口兼容性问题
- 影响：无法评估答案是否切题
- 解决方案：需要修复 RAGAS 与 LangChain 的兼容性

## 六、优化建议

### 短期优化（基于 RAGAS 发现）

**1. 提升上下文精度（目标：0.2 → 0.4）**
- 减少检索数量：k=6 → k=3
- 引入 Rerank 重排序：过滤掉不相关文档
- 优化文档切分：chunk_size 从 500 调整到 300

**2. 提升上下文召回（目标：0.4 → 0.6）**
- 补充知识库：添加缺失的文档
- 增加检索数量：在 Rerank 之前先检索更多（k=10）
- 优化 Embedding 模型：考虑使用中文优化的模型

**3. 提升忠实度（目标：0.5 → 0.7）**
- 优化 Prompt：强调"只基于文档回答"
- 添加二次验证：让 LLM 判断答案是否基于文档
- 降低 temperature：从 0.1 → 0.0

### 中期优化

**V3.0：Rerank 重排序**
- 预期效果：Context Precision 提升到 0.4+
- 实现方式：BGE Reranker

**V4.0：知识库优化**
- 补充缺失文档
- 优化文档结构和切分
- 预期效果：Context Recall 提升到 0.6+

## 七、技术问题记录

### 遇到的问题

1. **Answer Relevancy 评测失败**
   - 错误：`AttributeError: 'OpenAIEmbeddings' object has no attribute 'embed_query'`
   - 原因：RAGAS 0.4.3 与 langchain_openai 1.1.11 接口不兼容
   - 影响：无法评估答案相关性

2. **部分指标计算异常**
   - 警告：`LLM returned 1 generations instead of requested 3`
   - 影响：评测速度较慢，但结果仍然有效

### 解决方案

1. **短期**：使用现有的 3 个指标（Faithfulness, Context Precision, Context Recall）
2. **长期**：
   - 降级 langchain_openai 到兼容版本
   - 或等待 RAGAS 更新
   - 或使用自定义 Embedding 包装器
