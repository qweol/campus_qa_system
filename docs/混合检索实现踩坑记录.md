# RAG 混合检索实现踩坑记录

## 问题背景

在实现从 V1.0（纯向量检索）到 V2.0（混合检索）的升级过程中，遇到了一系列技术问题。这些问题的解决过程对理解 LangChain、RAG 系统有很大帮助。

## 问题1：LangChain 版本兼容性问题

### 问题描述

```python
from langchain.retrievers import EnsembleRetriever
# ModuleNotFoundError: No module named 'langchain.retrievers'
```

### 问题原因

LangChain 在不同版本中对模块结构进行了重构：
- 旧版本（0.1.x）：`from langchain.retrievers import EnsembleRetriever`
- 新版本（0.3.x）：`from langchain.retrievers.ensemble import EnsembleRetriever`

但即使修改了导入路径，仍然报错，说明当前安装的 LangChain 版本可能没有这个模块。

### 解决方案

**方案1（失败）**：尝试修改导入路径
```python
from langchain.retrievers.ensemble import EnsembleRetriever
# 仍然报错
```

**方案2（成功）**：自己实现混合检索器

```python
class HybridRetriever:
    """手动实现的混合检索器"""
    def __init__(self, vector_retriever, bm25_retriever):
        self.vector_retriever = vector_retriever
        self.bm25_retriever = bm25_retriever

    def invoke(self, query: str):
        # 向量检索
        vector_docs = self.vector_retriever.invoke(query)
        # BM25检索
        bm25_docs = self.bm25_retriever.invoke(query)
        # 合并去重
        seen_content = set()
        merged_docs = []
        for doc in vector_docs:
            content_hash = hash(doc.page_content[:100])
            if content_hash not in seen_content:
                seen_content.add(content_hash)
                merged_docs.append(doc)
        for doc in bm25_docs:
            content_hash = hash(doc.page_content[:100])
            if content_hash not in seen_content:
                seen_content.add(content_hash)
                merged_docs.append(doc)
        return merged_docs[:6]
```

### 经验总结

1. **依赖第三方库的风险**：框架更新快，API 经常变化
2. **自己实现的优势**：
   - 完全可控，不受版本限制
   - 可以自定义合并逻辑
   - 代码更简单，易于调试
3. **面试加分点**：展示了"遇到问题不依赖现成工具，自己实现"的能力

## 问题2：LCEL 管道类型不兼容

### 问题描述

```python
chain = (
    {"context": hybrid_retriever.invoke | _format_docs, ...}
    | prompt | llm | StrOutputParser()
)
# TypeError: unsupported operand type(s) for |: 'method' and 'function'
```

### 问题原因

LangChain 的 LCEL（LangChain Expression Language）使用 `|` 操作符连接组件，但要求：
- 左侧必须是 `Runnable` 对象
- `hybrid_retriever.invoke` 是一个普通的 Python 方法，不是 `Runnable`

### 解决方案

使用 `RunnableLambda` 包装普通函数：

```python
from langchain_core.runnables import RunnableLambda

chain = (
    {"context": RunnableLambda(hybrid_retriever.invoke) | _format_docs, ...}
    | prompt | llm | StrOutputParser()
)
```

### 经验总结

1. **LCEL 的设计理念**：所有组件都必须是 `Runnable`，才能用 `|` 连接
2. **RunnableLambda 的作用**：将普通函数转换为 `Runnable` 对象
3. **类似的包装器**：
   - `RunnablePassthrough`：直接传递输入
   - `RunnableLambda`：包装自定义函数
   - `RunnableParallel`：并行执行多个 Runnable

## 问题3：配置路径与实际项目结构不符

### 问题描述

```python
# app/config.py
vector_db_path: str = str(base_dir / "data" / "vectorstore")
# 但实际路径是: storage/faiss_index
```

### 问题原因

代码是从模板复制的，没有根据实际项目结构调整。

### 解决方案

```python
# 修改前
vector_db_path: str = str(base_dir / "data" / "vectorstore")
docs_path: str = str(base_dir / "data" / "docs")

# 修改后
vector_db_path: str = str(base_dir / "storage" / "faiss_index")
docs_path: str = str(base_dir / "data")
```

### 经验总结

1. **先检查再运行**：拿到代码后，先检查配置是否与实际环境匹配
2. **使用相对路径**：`Path(__file__).parent.parent` 确保路径相对于代码文件
3. **配置文件的重要性**：所有路径、参数都应该集中在配置文件中

## 问题4：文件格式不匹配

### 问题描述

```python
# BM25 加载器配置
loader = DirectoryLoader(
    settings.docs_path,
    glob="**/*.txt",  # 配置的是 .txt
    ...
)
# 但实际文件都是 .md 格式
```

### 问题原因

复制代码时没有检查实际文件格式。

### 解决方案

```python
loader = DirectoryLoader(
    settings.docs_path,
    glob="**/*.md",  # 改为 .md
    loader_cls=TextLoader,
    loader_kwargs={"encoding": "utf-8"}
)
```

### 经验总结

1. **先看数据再写代码**：了解数据格式、结构后再编码
2. **使用 `glob` 命令检查**：`ls data/*.md` 快速确认文件格式
3. **支持多格式**：生产环境应该支持 `.md`, `.txt`, `.pdf` 等多种格式

## 问题5：环境变量未加载

### 问题描述

```python
# 运行评测脚本
python scripts/evaluate_rag.py
# OpenAIError: The api_key client option must be set
```

### 问题原因

评测脚本没有加载 `.env` 文件，导致 `OPENAI_API_KEY` 为空。

### 解决方案

```python
# scripts/evaluate_rag.py
from dotenv import load_dotenv

# 在导入其他模块之前加载环境变量
load_dotenv()

from app.rag import build_rag_chain
```

### 经验总结

1. **环境变量加载顺序**：必须在使用配置之前调用 `load_dotenv()`
2. **不同运行方式的差异**：
   - `uvicorn app.main:app`：FastAPI 会自动加载 `.env`
   - `python scripts/xxx.py`：需要手动加载
3. **调试技巧**：`print(os.getenv("OPENAI_API_KEY"))` 快速检查环境变量

## 问题6：缺少依赖包

### 问题描述

```python
from langchain_community.retrievers import BM25Retriever
# 运行时报错：No module named 'rank_bm25'
```

### 问题原因

`BM25Retriever` 依赖 `rank-bm25` 包，但 `requirements.txt` 中没有。

### 解决方案

```bash
pip install rank-bm25
```

然后更新 `requirements.txt`：
```
rank-bm25==0.2.2
```

### 经验总结

1. **隐式依赖问题**：某些 LangChain 组件需要额外的依赖包
2. **依赖管理最佳实践**：
   - 每次安装新包后，立即更新 `requirements.txt`
   - 使用 `pip freeze > requirements.txt` 导出完整依赖
3. **文档不完善**：LangChain 文档没有明确说明 `BM25Retriever` 需要 `rank-bm25`

## 问题7：PYTHONPATH 未设置

### 问题描述

```bash
python scripts/evaluate_rag.py
# ModuleNotFoundError: No module named 'app'
```

### 问题原因

Python 默认只搜索当前目录和系统路径，不会搜索项目根目录。

### 解决方案

**方案1**：临时设置 PYTHONPATH
```bash
PYTHONPATH=/path/to/project python scripts/evaluate_rag.py
```

**方案2**：在脚本中动态添加路径
```python
import sys
from pathlib import Path
sys.path.insert(0, str(Path(__file__).parent.parent))
```

**方案3**：使用 `-m` 运行
```bash
python -m scripts.evaluate_rag
```

### 经验总结

1. **项目结构设计**：脚本应该放在项目根目录，或者使用包管理
2. **生产环境**：应该将项目打包成 Python 包（`setup.py` 或 `pyproject.toml`）
3. **开发环境**：使用 IDE（VS Code、PyCharm）会自动处理 PYTHONPATH

### 1. 依赖管理
